# 📰 news-crawler

**네이버 뉴스 키워드 크롤링 자동화 시스템**

---

## 📌 프로젝트 개요

- 매일 오전 9시, 네이버 뉴스에서 **지난 24시간 동안 자주 등장한 단어 목록을 수집**
- **등장 빈도 100회 이상 단어만 저장**하며, Oracle DB에 자동 저장
- 이전 데이터는 자동 삭제 (24시간 경과 시)

---

## 🧱 전체 시스템 설계

### 🎯 1. 시스템 목표
- 네이버 뉴스 검색 → 기사 본문 크롤링
- 텍스트 전처리 → 단어 빈도 분석
- 등장 빈도 100 이상 단어 Oracle DB 저장
- 매일 9시 자동 실행

### 🏗️ 2. 기능 구조

| 기능 영역            | 설명                                     |
|---------------------|------------------------------------------|
| 뉴스 크롤러         | Selenium으로 뉴스 제목/본문 수집         |
| 텍스트 전처리       | 불용어 제거, 특수문자 제거 등           |
| 단어 빈도 분석      | 등장 횟수 카운트 → 100회 이상 필터링    |
| DB 저장 모듈        | Oracle DB 저장                           |
| DB 클린업 모듈      | 24시간 지난 데이터 삭제                  |
| 스케줄러            | 매일 오전 9시 자동 실행                  |

## 📜 진행 로그

### 1. 원래 계획

- Java + Jsoup 로 네이버 뉴스 크롤링
- 기사 링크 → 본문 추출 → 분석 및 DB 저장
- 전체 파이프라인 자동화 (매일 9시)

### 2. 문제 발생

- Jsoup만으로는 네이버 뉴스 구조 파싱 불가
- 검색 결과가 JavaScript 기반으로 렌더링됨
- → Jsoup 방식 포기

### 3. 대안: Selenium 전환

- 브라우저 시뮬레이션 방식으로 크롤링 성공
- 뉴스 제목/본문 수집 완료 (실제 기사 본문 12개) -> 검색단어 : 인공지능
- 확인된 결과:
  - 링크 수집: 14개
  - 기사 본문 수집: 12개
